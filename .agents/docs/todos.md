## Runtime public API review

- [ ] Replace the current broad `App<TuiNode>` inheritance with a public `TuiApp` projection that tracks the consumer's installed Vue `App` surface automatically, replaces Vue's `mount`, and excludes every underscore-prefixed app field and the renderer-only `TuiNode`. Add clean-consumer declaration checks on supported Vue 3.4 and Vue 3.5 proving that ordinary plugin, provide, config, component, directive, unmount, and version-appropriate methods such as 3.5's `onUnmount()` appear while `_container` and other private host paths do not.
- [ ] Make `TuiApp.mount()` return the actual user root component instance, matching Vue's documented `app.mount()` result. The current implementation returns the private `InternalErrorBoundary` proxy instead.
- [ ] Change public `MountOptions.stdout` from `NodeJS.WriteStream` to Node's `Writable` from `node:stream`, while keeping omission mapped to `process.stdout`. Normalize optional terminal facts internally, remove unnecessary `PassThrough` and other ordinary-Writable casts from first-party helpers and tests, and add declaration and clean-consumer checks proving ordinary Node Writables are accepted without also accepting Web `WritableStream`.
- [ ] Change public `MountOptions.stdin` from `NodeJS.ReadStream` to Node's `Readable` and `MountOptions.stderr` from `NodeJS.WriteStream` to Node's `Writable`. Keep TTY-only facts and controls as optional internal capabilities, map omission to `process.stdin` and `process.stderr`, remove unnecessary ordinary-stream casts, and add declaration and clean-consumer checks.
- [ ] Align the accepted mount defaults and explicit Fullscreen capability gate. Omitted `mode` selects Inline, and an explicit Fullscreen request synchronously fails before application or terminal mutation when stdout is not a TTY or no positive terminal dimensions can be resolved. Do not silently fall back to Inline.
- [ ] Make a mount against an already-owned stdout throw synchronously before terminal or app state is mutated. The failed preflight must not consume the app, so it can be mounted after the existing owner releases the stream; remove the current warning, inert no-op, and placeholder component instance.
- [ ] Settle a consumed mount failure consistently: `mount()` throws the original error synchronously, Runtime completes resource rollback, and `waitUntilExit()` rejects with that same error instead of remaining pending.
- [ ] Align Inline non-TTY output with the vouched final-document contract. Do not emit screen-management sequences or every dynamic frame; append accepted Static history and coordinated console output immediately; on clean teardown write the current final dynamic document once, add a line ending only to non-empty output that lacks one, and write no bytes for an empty document. Do not replay a stale successful dynamic frame on error exit. Add focused pipe and file-like Writable tests for each distinction without inventing a public cadence override.
- [ ] Enforce borrowed ownership for mounted streams. Never call `end()` or `destroy()` on caller-supplied stdin, stdout, or stderr; remove Runtime listeners and restore only raw mode, ref state, terminal modes, and other state Runtime changed. Close only internally created resources. Add tests that reuse and explicitly close caller streams after `waitUntilExit()`.
- [ ] Route failures of actively used streams through the app lifecycle. Cover stdout-host loss; stdout and stderr synchronous write throws, callback failures including `EPIPE`, premature close during accepted writes or backpressure, stable Runtime errors for close without an `Error`, first-real-error precedence, active-input stdin loss versus input-free EOF, and a required final or restoration write failing after clean exit begins. Record the first fatal cause when observed, preserve it through later cleanup failures, and reject `waitUntilExit()` only after rollback and accepted output completion or abandonment; keep `waitUntilRenderFlush()` non-reporting.
- [ ] Make mount preflight and acquisition transactional. Before user setup or terminal mutation, resolve and validate options, stream defaults and protocol state, mode, stdout ownership, and Fullscreen TTY and dimensions. Then reserve stdout, establish reverse-order rollback, install stream observers and console protection, run user setup, validate stdin if setup creates active input demand, and only then acquire raw mode or other terminal resources and paint. Recheck capability when input demand later changes from inactive to active. Test every acquisition boundary and prove deterministic preflight failures do not consume the app.
- [ ] Change invalid untyped `useApp().exit(value)` behavior. Before teardown, a first non-`Error`, non-`undefined` value must synchronously throw `TypeError` without selecting or consuming exit; a caught error leaves the app running, while an uncaught error remains subject to surrounding Vue or input error handling. Calls after exit or teardown has started remain no-ops without argument validation. Replace the current conversion into a selected `TypeError` exit and add focused JavaScript-level tests.
- [ ] Align the app-owner barriers with the vouched lifecycle contract. Keep first-call-wins `exit(error?)` and `waitUntilExit()` settlement after teardown, restoration, and accepted output. Remove the current pre-mount and teardown availability errors. Add focused tests proving that a pre-mount barrier does not wait for a future mount, an active barrier waits for its accepted frame and stream callbacks, a teardown barrier waits and resolves after both clean and error exit, and a later application update is not implicitly included.
- [ ] Align `Static` with the vouched one-block contract.
