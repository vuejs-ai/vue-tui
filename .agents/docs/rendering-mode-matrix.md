# Rendering mode and host behavior matrix

> **Status:** unstamped F1.1 decision draft. This record describes the implementation at `f1ce02b` and the recommended target contract. It does not accept a public session API, mount-option migration, or implementation patch. The six product choices in [Maintainer decisions](#maintainer-decisions) remain open until explicitly accepted.

## Why the matrix has more than two rows

Inline and full-screen are the two live terminal screen models. They are not the only rendering surfaces vue-tui currently produces. A redirected stream, a screen-reader transcript, `renderToString`, and the content-frame testing harness do not acquire the same surface merely because the runtime internally falls through a boolean called `fullscreen`.

The matrix therefore keeps four questions separate:

1. **Render host:** live process, deterministic test, or synchronous string render.
2. **Requested mode:** inline or full-screen, when the host accepts a terminal-mode request.
3. **Effective surface:** main-screen live region, alternate-screen viewport, linear transcript, final stream output, static string, or test observation.
4. **Independent I/O facts:** live-update policy, stdout and stdin TTY state, screen-reader path, and terminal protocol availability.

This is a conceptual model, not a proposal to publish these exact names as one object. F1.2 and later F1 checkpoints own the mount shape and public readonly facts.

## Current resolution rules

The live renderer currently resolves its main branches as follows in [`render.ts`](../../packages/runtime/src/render.ts):

```ts
requestedFullscreen = Boolean(options.fullscreen ?? options.alternateScreen);
liveUpdates = options.interactive ?? (!isInCi && Boolean(stdout.isTTY));
alternateScreen = requestedFullscreen && liveUpdates && Boolean(stdout.isTTY);
fixedViewport = alternateScreen && !isScreenReaderEnabled;
targetedMousePath = alternateScreen && stdin.isTTY && terminalSupportsMouse;
```

`liveUpdates` is the truthful name for what the current `interactive` option primarily controls. It selects the live writer, resize listener, signal handler, Kitty initialization, and default lifetime raw-mode hold. It is not a complete statement that the application can or cannot receive input: current input composables can independently acquire raw mode on a TTY stdin even when `interactive: false`.

CI only changes the default for `interactive`; an explicit value wins. `fullscreen` wins over the deprecated `alternateScreen` alias whenever `fullscreen` is not `null` or `undefined`, including `fullscreen: false` with `alternateScreen: true`. An unavailable full-screen request is currently ignored without an error, warning, or observable requested-versus-effective fact.

## Current live-host matrix

These rows are exhaustive for the axes that select a rendering surface. `debug` and stdin behavior are finite modifiers covered immediately afterward.

| ID  | Requested mode | Current `interactive` result | stdout  | Screen reader | Current effective surface                                                                                                                                                                 |
| --- | -------------- | ---------------------------- | ------- | ------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| L1  | Either         | `false`                      | Either  | Either        | **Final-output stream.** No live terminal surface; new `Static` output is written as it appears and only the latest dynamic frame is written at teardown.                                 |
| L2  | Either         | `true`                       | Non-TTY | Off           | **Forced relative stream updater.** The inline writer emits updates to a stream, but there is no terminal screen, reliable physical origin, or hit map. A full-screen request is ignored. |
| L3  | Either         | `true`                       | Non-TTY | On            | **Forced transcript updater.** Linear screen-reader output is updated on the stream; a full-screen request is ignored.                                                                    |
| L4  | Inline         | `true`                       | TTY     | Off           | **Inline live region.** Relative updates on the main screen; completed output may move into terminal-owned history.                                                                       |
| L5  | Full-screen    | `true`                       | TTY     | Off           | **Fixed full-screen viewport.** Alternate screen, terminal-sized layout, origin `(0, 0)`, clipping, complete repaint, and a reliable hit-map coordinate system.                           |
| L6  | Inline         | `true`                       | TTY     | On            | **Main-screen transcript.** Linearized output with no fixed height, stable physical origin, or hit map.                                                                                   |
| L7  | Full-screen    | `true`                       | TTY     | On            | **Current hybrid defect.** The runtime enters the alternate screen and hides the cursor, but renders an unbounded linear transcript and builds no usable hit map.                         |

The normal default maps a non-CI TTY to L4-L7 according to the requested mode and screen-reader state, and a non-TTY or CI environment to L1. Forced `interactive: true` is the only route to L2 or L3.

### Debug modifier

`debug` does not participate in mode resolution, but it changes observation and output behavior:

| Base case    | Current debug behavior                                                                                                                                                                                                                                                 |
| ------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| L5           | The fixed viewport still clears, homes, and replaces the alternate-screen surface. The internal test frame sink also receives the content frame.                                                                                                                       |
| L1-L4, L6-L7 | Every commit appends the complete current content instead of maintaining the normal relative live region. In screen-reader cases the content is linearized; full `Static` history is replayed on later commits. L7 still enters the alternate screen before appending. |

Debug is an implementation and observation switch, not a third product rendering mode. The target contract must not require application code to infer terminal semantics from it.

### Development-reload modifier

An in-place Vue hot update keeps the mounted session and repaints through its existing surface. A Vite full reload runs the registered internal teardown before replacing the app, restoring terminal modes and the alternate screen without settling the old app's exit promise; the newly imported app then acquires its surface again. The target keeps that distinction: a hot update preserves the session, while a full reload must completely release and reacquire terminal ownership.

### Input and hit-map modifier

stdin is independent of the output surface:

| stdin and surface                           | Current behavior                                                                                                                                                                                                                    |
| ------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| stdin is non-TTY                            | Raw mode is unsupported. `useInput` and `useMouseInput` can throw while guarded focus behavior degrades; targeted mouse is unavailable.                                                                                             |
| stdin is TTY, stdout is non-TTY             | Input composables can acquire raw input even in L1-L3. Mouse and paste reporting cannot write their terminal control modes to the non-TTY stdout.                                                                                   |
| stdin and stdout are TTY, surface is not L5 | Terminal-wide input and `useMouseInput` can work, but there is no reliable element hit map. Inline mouse reporting still takes native selection and wheel behavior away from the terminal.                                          |
| stdin and stdout are TTY, surface is L5     | The current normal visual full-screen path can align terminal mouse coordinates with the clipped element hit map when the terminal supports SGR mouse.                                                                              |
| stdin and stdout are TTY, surface is L7     | Existing element handlers can arm SGR mouse reporting, but the screen-reader renderer produces no hit map, so the handlers cannot receive a target. This can suppress native mouse behavior without providing application behavior. |

## Current non-live hosts

| ID  | Host                                         | Current behavior                                                                                                                                                                                                                                                                                                                                                                                              |
| --- | -------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| H1  | Public `renderToString`                      | Synchronous width-constrained document, default 80 columns and unbounded height. It accepts no mode or rows, writes no terminal controls, prepends `Static` to dynamic output, and gives terminal composables mostly no-op contexts. It has no physical origin, viewport, history owner, hit map, resize, suspension, or final-output lifecycle.                                                              |
| H2  | Internal screen-reader string render         | H1 with linearized screen-reader output. It is not a public host choice, although JavaScript or `any` can currently pass the hidden option through the public function.                                                                                                                                                                                                                                       |
| H3  | `@vue-tui/testing` default                   | Fake TTY streams at 100×100, `debug: true`, and forced `interactive: true`. It observes content chunks from an implicit inline layout. `frames` and `lastFrame()` are not an emulated terminal screen, rows do not bound the layout, and there is no public option for full-screen or screen-reader behavior. The ambient `INK_SCREEN_READER` variable can still leak into the harness and change its output. |
| H4  | `@vue-tui/testing` with `interactive: false` | Still debug and still emits every content frame. It disables the runtime resize listener and lifetime raw-mode hold, but does not reproduce L1's final-frame-only production behavior.                                                                                                                                                                                                                        |

Two current leaks must not become target behavior: the string host's no-op app context exposes the real process streams, so `useWindowSize()` can observe and subscribe to the real stdout instead of the supplied virtual width, and a consumer can bypass the no-op `useStdout().write()` by writing directly to the exposed stream.

## Current surface behavior

Every live row maps to one surface class below, so lifecycle behavior does not need to be repeated in the resolution matrix.

| Surface                                 | History and owned region                                                                                                                                                                                                                               | `Static` and external output                                                                                                                                                                  | Resize                                                                                                                                             | Suspension                                                                                                            | Final output and restoration                                                                                                                                                           |
| --------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Final-output stream (L1)                | No live region. The stream owns emitted bytes.                                                                                                                                                                                                         | `Static` and coordinated stdout/stderr write immediately; the latest dynamic frame is deferred. Direct writes are indistinguishable from other stream writes.                                 | No runtime resize handler. Layout still resolves a width when rendering.                                                                           | No terminal session to restore, although input composables can currently acquire raw mode independently.              | Clean non-debug teardown writes the latest dynamic frame plus a newline. Non-interactive error output has separate behavior and can emit only the trailing newline.                    |
| Relative stream updater (L2-L3)         | No trustworthy terminal surface or history contract.                                                                                                                                                                                                   | Uses the live relative/transcript update paths against a non-TTY stream.                                                                                                                      | A listener is installed and reacts if the custom stream emits `resize`.                                                                            | No explicit support.                                                                                                  | The latest emitted bytes remain in the stream; no alternate-screen restoration occurs.                                                                                                 |
| Inline live region (L4)                 | The main screen and terminal own completed history; vue-tui can rewrite only its current relative dynamic region. The current overflow fallback can nevertheless emit `ESC[3J` and clear pre-app scrollback, then replay only retained `Static` bytes. | New `Static` output is written once above the dynamic region. Coordinated stdout/stderr clears and restores that region. Direct process-stream writes bypass coordination and can corrupt it. | Relayouts the current tree synchronously. Narrowing or viewport overflow can take the whole-screen clear path; completed history cannot be edited. | No `SIGTSTP`/`SIGCONT` model. An external stop can leave raw mode, cursor state, paste, Kitty, or mouse modes active. | The final current output remains on the main screen while terminal modes and cursor state are restored.                                                                                |
| Main-screen transcript (L6)             | Linear main-screen output; no bounded 2D surface.                                                                                                                                                                                                      | `Static` is emitted once. Dynamic transcript updates erase only their previous dynamic lines and avoid the whole-terminal clear path.                                                         | Rewraps to the current width; rows do not bound layout.                                                                                            | Same unsupported lifecycle as L4.                                                                                     | Transcript remains on the main screen and terminal modes are restored.                                                                                                                 |
| Fixed full-screen viewport (L5)         | vue-tui owns the current alternate-screen `columns × rows` viewport. The previous main screen remains owned by the terminal.                                                                                                                           | `Static` bytes and coordinated output remain stream-observable, then a clear/home repaint removes them from the visible viewport. Direct writes bypass the guarantee.                         | Recomputes rows and columns, clips, and repaints synchronously.                                                                                    | No explicit support; an external stop can leave the alternate screen and terminal modes active.                       | Teardown repaints once, restores terminal modes, leaves the alternate screen, and reveals the previous main screen. It does not replay the last frame or error overview after leaving. |
| Alternate-screen transcript hybrid (L7) | Alternate-screen lifetime with linear, height-unbounded transcript semantics. It has neither the fixed full-screen contract nor normal main-screen transcript history.                                                                                 | Screen-reader `Static` and output rules apply inside the alternate screen; targeted mouse can be armed without a usable hit map.                                                              | Rewraps by width; rows do not bound layout.                                                                                                        | Same unsupported lifecycle as L5.                                                                                     | Leaving the alternate screen discards the transcript and reveals the previous main screen.                                                                                             |

Normal unmount and app exit restore raw mode, bracketed paste, mouse reporting, Kitty keyboard state, cursor state, and the alternate screen. Interactive mounts also restore on SIGINT, SIGTERM, and SIGHUP. There is no suspend/resume implementation. A second live app on the same stdout is warned and ignored; direct process-stream writes and process-global console patching remain outside reliable multi-app coordination.

### Current exit-reason differences

| Exit reason                        | Inline or transcript                                                                                                                            | Fixed full-screen                                                                                                              | Final-output host                                                                              |
| ---------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------ | ---------------------------------------------------------------------------------------------- |
| Clean unmount or app exit          | The final current output remains on the main screen.                                                                                            | The alternate-screen frame disappears and no summary is replayed.                                                              | The latest dynamic frame plus a newline is written at teardown.                                |
| Fatal component error              | Interactive inline leaves the rendered ErrorOverview on the main screen.                                                                        | ErrorOverview is visible only in the alternate screen and disappears during restoration; nothing durable is printed afterward. | The current non-debug error path can emit only a trailing newline rather than a durable error. |
| SIGINT, SIGTERM, or SIGHUP         | Terminal modes are restored; no renderer summary is added.                                                                                      | Terminal modes and the previous main screen are restored; no frame or summary is replayed.                                     | No interactive signal handler is installed.                                                    |
| In-place HMR update or full reload | An update preserves the session. A full reload tears the old session down without resolving it as an app exit, then the replacement app mounts. | Same, including leaving and reacquiring the alternate screen on full reload.                                                   | Same teardown distinction when the dev host is active.                                         |

## Recommended target matrix

The target keeps inline and full-screen as the only terminal rendering modes while refusing to call every other output surface “inline.” “No effective mode” below means that no live terminal surface was acquired; it does not mean the application produced no useful output.

| Host and conditions                                            | Requested mode               | Recommended effective surface                         | Effective terminal mode                                             | Stable origin / hit map                                                            | Fallback or failure                                                                                                                                                 |
| -------------------------------------------------------------- | ---------------------------- | ----------------------------------------------------- | ------------------------------------------------------------------- | ---------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Live, live updates disabled, any stdout, either rendering path | Either                       | Final stream output                                   | None                                                                | No / no                                                                            | Preserve the request and expose that no terminal mode became effective; emit no rendering-surface control sequences. Input capabilities remain an independent fact. |
| Live, live updates enabled, visual path, non-TTY stdout        | Either                       | Relative stream updater                               | None                                                                | No / no                                                                            | Preserve for compatibility until F1.2 decides the option migration; never claim a terminal mode or targeted capability                                              |
| Live, live updates enabled, screen-reader path, non-TTY stdout | Either                       | Linear transcript updater                             | None                                                                | No / no                                                                            | Preserve for compatibility until F1.2 decides the option migration; never claim a terminal mode, alternate screen, or targeted capability                           |
| Live, live updates enabled, visual TTY                         | Inline                       | Main-screen live region                               | Inline                                                              | No / no                                                                            | None                                                                                                                                                                |
| Live, live updates enabled, visual TTY                         | Full-screen                  | Fixed alternate-screen viewport                       | Full-screen                                                         | Yes / available only when the independent input and terminal capabilities allow it | None                                                                                                                                                                |
| Live, live updates enabled, screen-reader TTY                  | Inline                       | Main-screen linear transcript                         | Inline                                                              | No / no                                                                            | None                                                                                                                                                                |
| Live, live updates enabled, screen-reader TTY                  | Full-screen                  | Main-screen linear transcript                         | Inline                                                              | No / no                                                                            | Detectable fallback; never enter alternate screen or arm targeted mouse                                                                                             |
| Synchronous string host                                        | Not accepted                 | Static width-constrained document                     | None                                                                | No / no                                                                            | Mode is a type-level non-option; add a bounded rows×columns snapshot only as a separately justified API                                                             |
| Deterministic test host                                        | Explicit modeled environment | Content observation plus the selected modeled surface | The modeled inline/full-screen mode, or none for final-output tests | Derived from the modeled surface                                                   | Test controls choose from the finite production classes instead of exposing arbitrary internal booleans                                                             |

The test host should expose content frames and an emulated final terminal surface as different observations. `interactive: false` must either reproduce the production final-output class or be replaced by a control whose narrower meaning is truthful. Static and test virtual dimensions must be supplied by their host rather than leaking the developer's real process streams.

Fallback only describes an environment in which the requested screen model cannot exist. It does not make inline primary or full-screen degraded; both remain first-class when their actual terminal surfaces are available.

### Recommended lifecycle contract

| Surface                    | Target promise                                                                                                                                                                                                                                                                                                                                                                                                                                     |
| -------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Inline live region         | Never erase terminal history or shell output that existed before the app. `Static` transfers completed output to terminal-owned history. The redrawable dynamic region is bounded to the terminal's addressable rows; ordinary content beyond that bound follows explicit layout clipping instead of pushing mutable rows into history. Coordinated output may move the region but must re-establish it without claiming a stable physical origin. |
| Fixed full-screen viewport | Keep the merged fixed viewport, clipping, coordinated repaint, and direct-write boundary. `Static` remains non-persistent on the visible surface.                                                                                                                                                                                                                                                                                                  |
| Main-screen transcript     | Remain linear and main-screen based. Never enter the alternate screen or arm targeted mouse merely because full-screen was requested.                                                                                                                                                                                                                                                                                                              |
| Suspension                 | On external `SIGTSTP`, restore raw, mouse, paste, Kitty, cursor, and alternate-screen state before the process stops. After `SIGCONT`, re-read dimensions and re-establish only the effective surface: redraw the inline live region or re-enter and repaint the full-screen viewport. Suspension emits no final summary. Whether vue-tui reserves the Ctrl+Z input byte and initiates suspension is a later input-routing decision.               |
| Final output               | Inline and transcript output remains on the main screen. Full-screen restores the previous screen and prints no final frame automatically. Final-output hosts emit `Static` plus the latest dynamic result. An application summary after full-screen exit is explicit application output, not an implicit renderer replay.                                                                                                                         |
| Fatal error                | A fatal error must remain observable after terminal restoration. Inline can leave its ErrorOverview on the main screen; full-screen restores its terminal state before emitting a durable error, and final-output hosts emit the error directly to stderr. Exact formatting and host override belong to the public error contract.                                                                                                                 |

## Finite session facts

F1 needs enough observable state for later APIs to tell the truth, but it must not predesign pointer, focus, or input-routing operations. The finite facts implied by this matrix are:

- render host: live, deterministic test, or synchronous string;
- requested terminal mode when the host accepts one;
- effective terminal mode or no acquired terminal mode;
- effective surface class and any fallback reason;
- live-update policy, separately from stdin availability;
- screen-reader transcript state;
- stdout TTY, stdin TTY, and raw-input support as independent facts;
- columns and rows plus whether rows bound the current layout;
- stable physical origin and element hit-map availability;
- history owner, final-output policy, and suspension support.

The current `interactive` boolean must not be published as a broad “the application is interactive” fact: it does not gate all input composables today. Exact property names, whether the facts are one discriminated readonly object or several focused composables, and how capability negotiation reports unknown terminal support belong to later F1 checkpoints.

## Maintainer decisions

The source and existing vouched records determine the rest of the matrix. These are the remaining product choices:

1. **Unavailable full-screen request outside screen-reader rendering:** accept clean, detectable fallback rather than throw or print a warning. A non-TTY or final-output host acquires no terminal mode. **Recommendation: accept.** Redirected CLI output stays useful and warnings do not pollute pipelines, while application code can still observe the fallback.
2. **Inline ownership and overflow:** promise that vue-tui never clears terminal history or shell output from before the app started, and bound the mutable live region to addressable terminal rows. `Static` is the explicit operation that commits completed rows to terminal history; ordinary dynamic overflow is clipped by layout instead of pretending that rows already scrolled into history remain mutable. The current `ESC[3J` path becomes an implementation defect to replace. **Recommendation: accept.** This preserves unrelated history and gives arbitrary Vue updates an honest redraw boundary; append-only duplicate snapshots would not preserve component semantics.
3. **Screen reader with a full-screen request:** use the main-screen transcript, do not enter the alternate screen, and do not arm targeted mouse. **Recommendation: accept.** The current hybrid provides neither full-screen geometry nor durable transcript behavior and can suppress native mouse interaction for no usable event.
4. **Suspend/resume:** include external `SIGTSTP`/`SIGCONT` in the mature lifecycle contract with restore-before-stop and re-establish-after-continue behavior. **Recommendation: accept as a target requirement.** Whether Ctrl+Z is a default framework shortcut belongs to F3 input routing; F1 only defines behavior once the process is suspending or continuing.
5. **Default exit output:** keep the natural screen-owner behavior: inline and transcript leave their output on the main screen; full-screen restores the previous screen and prints no automatic final frame. **Recommendation: accept.** Applications that need a shell-visible summary can explicitly produce one after the session exits.
6. **Fatal error output:** require a durable error after restoration rather than applying the clean-exit “no final frame” rule. Inline can retain ErrorOverview; full-screen restores first and then writes the error to stderr, while final-output hosts write it directly. **Recommendation: accept.** The existing full-screen and non-interactive paths can otherwise erase the only explanation of why the application stopped.

The host split, independent input facts, static-host behavior, and separate test observations are consequences of the existing terminal models rather than additional product preferences. The exact public API and compatibility schedule remain F1.2 work after these six choices are accepted.
