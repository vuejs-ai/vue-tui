# Runtime foundation summary

This records the final public package boundary and evidence for the minimum Runtime foundation developed on PR #265.

> **Current implementation:** The accepted public root and `/inline` contracts are implemented. Official Vite and testing bridges now use narrow unsupported internal entries; the former public-looking `/devtools` and `/testing` package paths are absent.

The PR re-derived `@vue-tui/runtime` as the minimum public set of terminal and renderer primitives that only Runtime can provide. Earlier broad focus routing, geometry, caret, pointer, selection, and clipboard experiments were removed after the public boundary settled; only the narrow rendered-target, Box-metrics, focus, input, output, and lifecycle mechanisms required by accepted contracts remain.

The authoritative acceptance record is the [Runtime public API decision ledger](./runtime-public-api-decisions.md). The [public-foundation re-audit](./runtime-public-foundation-reaudit.md) explains the complete user paths and implementation evidence. This summary only makes their final result easy to inspect; it does not introduce another API authority.

## Result to review

The final common authoring entry exposes exactly these values:

```ts
import {
  Box,
  Text,
  createApp,
  renderToString,
  useApp,
  useBoxMetrics,
  useFocus,
  useInput,
  useLayoutSize,
  useStdin,
} from "@vue-tui/runtime";
```

The root also exports only the named user-consumable types associated with those values: `TuiApp`, `MountOptions`, `RenderToStringOptions`, `BoxProps`, `Color`, `TextProps`, `UseAppReturn`, `FocusTarget`, `UseFocusReturn`, `TuiInputEvent`, `TuiKey`, `TuiKeyName`, `UseStdinReturn`, `UseLayoutSizeReturn`, and `UseBoxMetricsReturn`.

The selected supported subpaths are deliberately small:

| Package entry                   | Public values    | Intended consumer                          | Why it is public                                                                                                           |
| ------------------------------- | ---------------- | ------------------------------------------ | -------------------------------------------------------------------------------------------------------------------------- |
| `@vue-tui/runtime/inline`       | `Static`         | Applications using Inline terminal history | Runtime alone can turn a mounted Vue slot tree into irreversible terminal history.                                         |
| `@vue-tui/runtime/package.json` | Package metadata | Version-matched tooling and guide lookup   | Consumers can resolve the shipped Runtime version and visual-development guide without depending on private `dist` layout. |

The official `@vue-tui/vite` and `@vue-tui/testing` packages remain public products, but their connections to Runtime are privileged implementation details. They use `@vue-tui/runtime/internal/devtools` and `@vue-tui/runtime/internal/testing`; a third party can technically import these entries because they must ship for Node resolution, but Runtime gives them no public compatibility promise.

Private implementation modules do not need to be mechanically re-exported from one `internal.ts` barrel; privacy means that they remain outside every supported package entry. `/fullscreen` is also absent from the selected public surface because Fullscreen is a mount mode rather than a separate component or composable universe.

## What changed from `main` and the earlier PR direction

| Area                     | Earlier surface or experiment                                                                                                                                                     | Final public contract                                                                                                                                                                                                                              |
| ------------------------ | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Rendering                | Root `Static`, `Newline`, `Spacer`, `Transform`, and `useAnimation` mixed primitives with conveniences.                                                                           | Root keeps `Box` and `Text`; `Static` moves to `/inline`; newlines and flex spacers are ordinary composition, while transform and animation remain above the minimum until concrete use justifies a stable contract.                               |
| App surface              | Vue's complete renderer `App<TuiNode>` type leaked underscore-prefixed fields and the private host type; a hidden component boundary changed the mount result and Vue error flow. | `TuiApp` preserves the consumer's public Vue application operations, excludes private fields and `TuiNode`, returns the actual user root, follows Vue component error handling, and adds only terminal mount plus the two completion barriers.     |
| Layout facts             | `useRenderSession`, physical-window projections, `measureElement`, and experimental general geometry exposed host and renderer structure.                                         | `useLayoutSize` exposes one numeric root-layout snapshot, and direct-Box `useBoxMetrics` exposes one parent-relative accepted rectangle.                                                                                                           |
| Input                    | String-plus-`Key`, separate `usePaste`, parser details, route results, availability, focused/scope/external hooks, and protocol controls mixed facts with routing policy.         | `useInput` broadcasts tagged `text`, `key`, and `paste` facts; `useStdin` is the complete low-level stream and independently owned raw-mode escape; focus and higher routing compose above them.                                                   |
| Focus                    | Manager access, string IDs, traversal, automatic Tab, scopes, traps, restoration, and automatic-focus policy were public experiments.                                             | Two `useFocus` overloads create one explicit unique owner, optionally tied to a real component ref's rendered validity; Runtime publishes no navigation policy.                                                                                    |
| Output and lifecycle     | Direct stdout/stderr composables, coordinated write results, render cadence, screen-reader presentation, and broad mount controls exposed scheduler or host policy.               | Runtime owns streams, console protection, terminal acquisition, restoration, first-cause failure, Inline/Fullscreen selection, and non-TTY final output behind `MountOptions`, `useApp().exit()`, `waitUntilRenderFlush()`, and `waitUntilExit()`. |
| Advanced interaction     | Physical caret, target geometry, mouse routing, drag capture, arbitrary painted-text selection, and OSC 52 clipboard were implemented together as broad F5–F8 surfaces.           | Those public surfaces and their unused controllers and services are removed. A future narrow primitive must be reintroduced only after a concrete application proves the current public foundation insufficient.                                   |
| Accessibility experiment | Screen-reader presentation, ARIA props and types, environment selection, transcript rendering, and test-only selectors formed an incomplete second rendering contract.            | The experiment is removed rather than presented as accessibility support. A future proposal needs its own complete semantics and evidence.                                                                                                         |

## Explicitly outside this foundation

- No `@vue-tui/use` package is created by this PR. A replaceable higher layer should grow only after real applications repeat behavior worth sharing.
- No physical caret, pointer targeting, arbitrary painted-text selection, OSC 52 clipboard, child-PTY forwarding, or generic terminal-protocol transaction is promised.
- No screen-reader or ARIA contract is claimed.
- No component catalog, formatted error screen, application policy, merge decision, release, or 1.0 compatibility promise is part of this foundation.
- Private implementation breadth is not a public API defect by itself. Reopen it only for correctness, measured performance, maintainability, or a concrete public capability.

## Completion boundary

The Runtime public foundation is complete for this phase. No public Runtime API design remains open. Official Vite and testing bridges are privileged internal implementation channels, not third-party contracts. Future caret, pointer, selection, clipboard, screen-reader, `@vue-tui/use`, and component work is outside this foundation.

The minimum Runtime foundation is complete when the exact public exports, their user-visible contracts, package-boundary checks, representative in-repository examples, fixtures, and clean consumers, and terminal lifecycle evidence agree. The Runtime root, `/inline`, host behavior, ownership model, and narrow official-tooling entries now match the accepted contract. No new Runtime API design is required. Future Runtime APIs require new application evidence and a separate accepted decision; the expected next work is application behavior, optional public-only higher layers, and components.

This conclusion does not merge or release PR #265. Those remain later maintainer decisions.
